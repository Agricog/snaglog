# SnagLog - Project Specification

**Domain:** snaglog.co.uk  
**Tagline:** "Photos In. Snag List Out."  
**Model:** DetailRecon-style (AI photo analysis â†’ professional PDF report)

---

## The Problem

Snagging a new build property currently takes:
- 2-4 hours walking the property with clipboard
- Photographing 50-150 defects
- Manually categorizing by room/trade/severity
- Writing up descriptions and remedial actions
- Formatting into presentable report for developer

**Result:** Tedious, time-consuming, inconsistent quality

---

## The Solution

1. User uploads photos of defects (batch upload)
2. AI analyzes each image - identifies defect type, severity, likely trade responsible
3. User confirms/edits room assignments
4. System generates professional PDF snag list

**Time saved:** 2-4 hours â†’ 15-20 minutes

## AI Analysis Requirements

### What AI Must Detect

**Defect Categories:**
- Paint defects (scuffs, missed spots, runs, uneven coverage)
- Plaster/drywall issues (cracks, dents, unfinished areas)
- Joinery defects (gaps, damage, poor fitting)
- Flooring issues (scratches, gaps, lifting)
- Tile problems (cracks, grout issues, uneven)
- Window/door issues (seals, mechanisms, damage)
- Kitchen fitting (alignment, damage, incomplete)
- Bathroom fitting (seals, fixtures, grouting)
- Electrical (unfinished plates, damage, alignment)
- External (pointing, drainage, finishes)

### AI Output Per Photo

```json
{
  "defect_type": "Paint - scuff mark",
  "description": "Scuff mark on wall approximately 30cm from floor, likely from construction traffic",
  "severity": "minor",
  "suggested_trade": "Decorator",
  "remedial_action": "Touch up paint required",
  "confidence": 0.92
}
```

### AI Model Options

1. **Claude Vision API** - Your current stack, consistent with DetailRecon
2. **GPT-4 Vision** - Alternative, similar capability
3. **Custom fine-tuned model** - Future optimization

**Recommendation:** Start with Claude Vision (consistency with DetailRecon workflow)

## Database Schema (MVP)

```sql
-- Reports
CREATE TABLE reports (
  id UUID PRIMARY KEY,
  property_address TEXT NOT NULL,
  property_type VARCHAR(50),
  developer_name TEXT,
  inspection_date DATE,
  created_at TIMESTAMP DEFAULT NOW(),
  status VARCHAR(20) DEFAULT 'draft',
  payment_status VARCHAR(20) DEFAULT 'unpaid',
  stripe_payment_id TEXT,
  pdf_url TEXT
);

-- Snags
CREATE TABLE snags (
  id UUID PRIMARY KEY,
  report_id UUID REFERENCES reports(id),
  photo_url TEXT NOT NULL,
  room VARCHAR(100),
  defect_type VARCHAR(100),
  description TEXT,
  severity VARCHAR(20),
  suggested_trade VARCHAR(100),
  remedial_action TEXT,
  ai_confidence DECIMAL(3,2),
  user_edited BOOLEAN DEFAULT FALSE,
  display_order INTEGER
);
```
